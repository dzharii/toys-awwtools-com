# HUMANS.md

Detailed handoff document for the animated meeting timeline prototype in this directory.

This document is written for the next engineer or designer-agent who needs to continue the project without rediscovering the same constraints, tooling, and implementation details.

## Current Project State

This is a static, backend-free web app for an animated, editable, shareable meeting timeline.

Runtime files:
- `index.html`: semantic page structure and controls.
- `style.css`: full visual layout, timeline geometry, layering, responsive behavior, CSS character fallback, and sprite frame display.
- `script.js`: state, URL hash parsing/serialization, time-zone conversion, edit behavior, countdown journey math, rendering, copy-link behavior, and animation loop.
- `assets/sprites/atlases/*.png`: cleaned production sprite atlases used at runtime.
- `assets/sprites/sprites.json`: sprite metadata generated by the slicer.

Source/spec/reference files:
- `suggestions001.md`: primary product and behavior specification.
- `website-design.png`: primary visual reference. Use this for layout, spacing, card hierarchy, timeline proportions, and product polish.
- `img-source/*.png`: generated source sprite sheets. These are not production-ready UI assets and must not be displayed directly.

Build/tooling files:
- `scripts/slice-sprites.mjs`: repeatable ImageMagick-based sprite slicing pipeline.
- `IMPLEMENTATION_PLAN.md`: short implementation plan from the first build.
- `IMPLEMENTATION_REPORT.md`: current report of repairs, model choices, test results, and known limitations.

Archives:
- `__archived/`: earlier prototype attempts and source copies. Do not edit for current runtime unless intentionally comparing behavior.

## Core Product Model

The meeting timeline has one canonical meeting instant.

The app displays that same instant in:
- UTC.
- Participant A time zone.
- Participant B time zone.

The important timeline repair was adding a separate creation instant:
- `at`: meeting start time in UTC.
- `ct`: countdown creation time in UTC.

The character journey is anchored by `ct -> at`, not by a generic display bucket.

Correct progress formula:

```js
createdAtMs = Date.parse(config.createdAtUtc)
meetingAtMs = Date.parse(config.startsAtUtc)
nowMs = Date.now()
journeyMs = meetingAtMs - createdAtMs
elapsedMs = nowMs - createdAtMs
progressRatio = clamp(elapsedMs / journeyMs, 0, 1)
remainingRatio = 1 - progressRatio
```

Participant positions use one shared progress value:

```js
maxOffsetPercent = 36
participantA.x = 50 - remainingRatio * maxOffsetPercent
participantB.x = 50 + remainingRatio * maxOffsetPercent
```

Sprite frame selection uses the same progress value:

```js
frameIndex = clamp(Math.round(progressRatio * 31), 0, 31)
column = frameIndex % 8
row = Math.floor(frameIndex / 8)
```

This is deliberate. Do not return to using adaptive display windows as character progress.

## URL Hash Format

Stable key order used by serializer:

```text
#v=1&t=Design%20Sync&at=2026-05-26T08%3A45Z&ct=2026-05-23T06%3A36Z&dur=60&a=Alice@America~Los_Angeles&b=Bruno@Europe~Berlin&theme=light&anim=1&mode=view&render=sprite&spriteA=female-minimal-soft&spriteB=male-dark-precision
```

Keys:
- `v`: schema version.
- `t`: meeting title.
- `at`: UTC meeting start.
- `ct`: UTC countdown creation time.
- `dur`: duration in minutes.
- `a`: Participant A name and timezone using `name@timezone`, with `/` replaced by `~`.
- `b`: Participant B name and timezone using `name@timezone`, with `/` replaced by `~`.
- `theme`: `light`, `neutral`, `dark`, or `auto`.
- `anim`: `1` or `0`.
- `mode`: `view` or `edit`.
- `render`: `sprite` or `svg`.
- `spriteA`: selected Participant A sprite key.
- `spriteB`: selected Participant B sprite key.

Backward compatibility:
- Old links without `ct` are migrated by generating `ct` once and replacing the hash.
- Existing `ct` must be preserved across reloads and copy-link.

## Time-Zone Editing Rules

Changing a participant timezone preserves the meeting instant by default.

Example:
- Meeting UTC is `18:00`.
- Bruno timezone changes from `Europe/Berlin` to `Asia/Tokyo`.
- UTC stays `18:00`.
- Alice equivalent time stays equivalent to UTC.
- Bruno local time changes to Tokyo local time.
- Hash changes from `b=Bruno@Europe~Berlin` to `b=Bruno@Asia~Tokyo`.

Editing a time row changes the canonical instant:
- UTC row is source: Alice and Bruno rows derive from UTC.
- Alice row is source: UTC and Bruno rows derive from Alice local time.
- Bruno row is source: UTC and Alice rows derive from Bruno local time.

Important UX detail:
- Inputs that currently have focus are not overwritten during render.
- This is what makes timezone text fields practically editable.

Known limitation:
- DST ambiguous times choose the earliest matching instant.
- Nonexistent local times show validation, but nearest-time suggestions are not implemented.

## Sprite System

Do not display any `img-source/*.png` directly in the UI.

Those files are wide generated sprite sheets with title text, frame numbers, grid lines, and inconsistent whitespace. Rendering them raw is a hard visual failure.

Runtime uses cleaned atlas files:

```text
assets/sprites/atlases/female-dark-precision.png
assets/sprites/atlases/female-enterprise-friendly.png
assets/sprites/atlases/female-glass-focus.png
assets/sprites/atlases/female-minimal-soft.png
assets/sprites/atlases/female-storyboard-warmth.png
assets/sprites/atlases/male-calendar-companion.png
assets/sprites/atlases/male-dark-precision.png
assets/sprites/atlases/male-minimal-soft.png
assets/sprites/atlases/male-premium-command.png
assets/sprites/atlases/male-storyboard-warmth.png
```

Each production atlas is:
- `2048x1024` pixels.
- 8 columns x 4 rows.
- 32 frames.
- 256x256 per frame.

Runtime CSS displays exactly one frame:

```css
.sprite-frame {
  width: 256px;
  height: 256px;
  background-size: 2048px 1024px;
}
```

Runtime JS sets:

```js
background-position: `${-col * 256}px ${-row * 256}px`
```

The slicer includes repair behavior for male sheets:
- Some source male bottom-row frames are clipped at the source image boundary.
- The slicer remaps male bottom-row production frames from a safer reaching row and applies crop offsets to keep full-body frames usable.
- This is documented in `scripts/slice-sprites.mjs`.

## Visual Layout Notes

The target is `website-design.png`.

The current app no longer has the original left sidebar because the user explicitly asked to remove it and unnecessary chrome. However, the main screen should still borrow the reference's product quality:
- Large clean hero card.
- Center meeting card above the line.
- Countdown pill below the center card.
- Side participant cards above sprites.
- Horizontal timeline axis and ticks.
- Polished top controls.
- Bottom summary strip.

Layering strategy:
- Decorative background/orbs: lowest.
- Timeline axis and connector line: passive lower layers.
- Sprites: above timeline/ticks.
- Meeting card, countdown pill, and participant cards: above sprites where needed for readability.
- Edit panel: normal document flow below timeline.

Important CSS selectors:
- `.timeline-card`: outer hero surface.
- `.timeline-stage`: absolute-positioned timeline canvas.
- `.meeting-card`: center meeting information card.
- `.participant-card`: side name/time cards, hoisted above sprites.
- `.character`: participant position container.
- `.sprite-frame`: one atlas frame.
- `.timeline-axis`, `.tick`, `.center-marker`: timeline marks.
- `.edit-panel`: edit mode form.

## Headless Browser Regression Script

This project does not currently commit a test framework. The following ad hoc command was useful for verifying real browser behavior.

Prerequisite if `/tmp/meettime-test` does not exist:

```bash
mkdir -p /tmp/meettime-test
npm install --prefix /tmp/meettime-test puppeteer-core
```

Run local server first:

```bash
python3 -m http.server 5500
```

Then run a regression check:

```bash
node - <<'NODE'
const puppeteer = require('/tmp/meettime-test/node_modules/puppeteer-core');
const executablePath = '/home/dzhariy/.cache/ms-playwright/chromium-1223/chrome-linux64/chrome';
const isoMinute = ms => new Date(ms).toISOString().replace(/:00\.000Z$/, 'Z');
(async () => {
  const browser = await puppeteer.launch({ executablePath, headless: 'new', args: ['--no-sandbox'] });
  const page = await browser.newPage();
  await page.setViewport({ width: 1440, height: 900 });
  const errors = [];
  page.on('console', msg => { if (msg.type() === 'error') errors.push(msg.text()); });
  page.on('pageerror', err => errors.push(err.message));
  const assert = (ok, msg) => { if (!ok) throw new Error(msg); };
  const text = sel => page.$eval(sel, el => el.textContent.trim());
  const value = sel => page.$eval(sel, el => el.value);
  const hash = () => page.evaluate(() => window.location.hash);
  const styleVar = name => page.$eval('#timelineStage', (el, name) => getComputedStyle(el).getPropertyValue(name).trim(), name);
  const setInput = (sel, v) => page.$eval(sel, (el, v) => { el.value = v; el.dispatchEvent(new Event('input', { bubbles: true })); }, v);

  await page.goto('http://127.0.0.1:5500/', { waitUntil: 'networkidle0' });
  let h = await hash();
  assert(h.includes('at='), 'default hash missing at');
  assert(h.includes('ct='), 'default hash missing ct');
  const initialCt = decodeURIComponent(h.match(/ct=([^&]+)/)[1]);
  await page.reload({ waitUntil: 'networkidle0' });
  assert(decodeURIComponent((await hash()).match(/ct=([^&]+)/)[1]) === initialCt, 'ct not preserved on reload');

  const now = Date.now();
  const at2d = isoMinute(now + 2 * 24 * 60 * 60 * 1000);
  const ctNow = isoMinute(now);
  await page.goto(`http://127.0.0.1:5500/#v=1&t=Two%20Day&at=${encodeURIComponent(at2d)}&ct=${encodeURIComponent(ctNow)}&dur=60&a=Alice@America~Los_Angeles&b=Bruno@Europe~Berlin&theme=light&anim=1&mode=view&render=sprite&spriteA=female-minimal-soft&spriteB=male-dark-precision`, { waitUntil: 'networkidle0' });
  const aStart = parseFloat(await styleVar('--participant-a-x'));
  const bStart = parseFloat(await styleVar('--participant-b-x'));
  assert(aStart < 18 && bStart > 82, `2-day journey did not start near edges: ${aStart}, ${bStart}`);

  await page.click('#editButton');
  await page.waitForSelector('#editPanel:not([hidden])');
  await setInput('#bTzInput', 'Asia/Tokyo');
  await new Promise(r => setTimeout(r, 450));
  assert(await value('#bTzInput') === 'Asia/Tokyo', 'Bruno timezone input not editable');
  assert((await hash()).includes('b=Bruno@Asia~Tokyo'), 'timezone hash did not update to Tokyo');

  await setInput('#utcDateInput', '2026-05-24');
  await setInput('#utcTimeInput', '18:30');
  await new Promise(r => setTimeout(r, 350));
  assert(await value('#aTimeInput') === '11:30', 'UTC edit did not update Alice');

  assert(errors.length === 0, 'browser console errors: ' + errors.join('\n'));
  await browser.close();
  console.log('regression smoke passed');
})();
NODE
```

This is intentionally a smoke version. The fuller suite used during repair also tested cancel/save/copy/mobile.

## Common Pitfalls

Do not use `remainingMs / adaptiveWindowMs` for character positions.

Do not remove or reset `ct` during reload, save, or copy.

Do not use raw `img-source/*.png` as a character image.

Do not set sprite frame CSS so `background-size` differs from the full atlas scaled dimensions.

Do not overwrite focused text inputs during render. This makes timezone editing feel broken.

Do not place participant cards low enough to overlap walking sprites.

Do not use `object-fit: cover` or clipping containers for sprites.

Do not assume all source atlases are equally clean. The slicer has repair logic for a reason.
