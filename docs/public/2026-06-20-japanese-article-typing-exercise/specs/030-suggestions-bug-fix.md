E00 Follow-Up Change Notes - Meaning Line, UI Hints, Favicon, Defaults, and Hotkeys

---

This document describes the follow-up changes made after `020-suggestions-bug-fix.md`.

The goal of this pass was to improve clarity and safety in the practice experience without changing the core guided typing model from the 020 bug-fix pass. The changes focus on visible English support for the active sentence, discoverable UI labels, app identity, more realistic default pacing, and preventing letter keys from triggering app commands while the learner is typing.

E02 CHG-001 - Active English Sentence Line

---

The practice view now displays a quiet English sentence line for the current active sentence.

Implementation:

```html
<div id="active-meaning-line" class="active-meaning-line"></div>
```

The line is rendered below the romaji input field, not directly under the magnified Japanese text. This keeps the primary focus on the Japanese sentence and typing input, while still giving the learner an English reference in the active writing area.

The English text is generated by joining the `meaning` value from each token in the active sentence:

```js
function buildMeaningSentence(sentence) {
  return sentence.tokens
    .map(token => token.meaning)
    .filter(Boolean)
    .join(" ");
}
```

Behavior:

| Condition                         | Result                                      |
| --------------------------------- | ------------------------------------------- |
| Active sentence tokens have meanings | Meanings are joined and shown below input. |
| Some tokens have no meaning       | Empty meanings are skipped.                 |
| No token has a meaning            | The English sentence line is hidden.        |

E03 CHG-002 - Hover Hint Text for UI Labels

---

Visible labels and controls now include native browser hover hints through `title` attributes. These hints include an English translation and a short explanation of what the item does.

Examples:

| UI text       | Hint intent                                      |
| ------------- | ------------------------------------------------ |
| `進捗`        | Explains progress and article completion.        |
| `時間`        | Explains active practice time.                   |
| `文字数`      | Explains completed characters out of total text. |
| `音`          | Explains sentence speech playback.               |
| `現在の文`    | Explains the active sentence area.               |
| `前の文`      | Explains the previous-sentence context area.     |
| `再開`        | Explains that resume restarts timer and pacing.  |
| `一時停止`    | Explains that pause stops timer and pacing.      |

Generated UI also received hints:

| Area             | Coverage                                                     |
| ---------------- | ------------------------------------------------------------ |
| Menu actions     | Resume, restart sentence, reset progress, import, export.    |
| Settings         | Group headings, rows, inputs, selects, toggles, mini buttons.|
| Report           | Summary cards, token section headings, table columns.        |
| Dialogs          | Dialog titles, messages, and action buttons.                 |
| Drop overlay     | Drag-and-drop import status text.                            |

The hints are intentionally simple browser-native hints, not custom tooltips. This avoids adding layout complexity or visual noise to the quiet practice surface.

E04 CHG-003 - Favicon

---

The app now includes a local favicon:

```txt
favicon.ico
```

The page head links it with:

```html
<link rel="icon" href="favicon.ico" sizes="any" />
```

The icon was generated with ImageMagick using the app's paper-like colors and the `禅` seal motif. The generated ICO includes common sizes:

```txt
256, 128, 64, 48, 32, 16
```

This keeps the static GitHub Pages deployment self-contained and avoids any external favicon service.

E05 CHG-004 - Practice Delay Defaults

---

The default practice delay settings were updated:

| Delay  | New default |
| ------ | ----------: |
| Short  | 2400 ms     |
| Medium | 6400 ms     |
| Long   | 10000 ms    |

Implementation:

```js
delays: { short: 2400, medium: 6400, long: 10000 }
```

No saved-settings migration is included. The change updates the application defaults only. Existing browsers with saved custom settings keep their saved values until the user changes or resets settings.

E06 CHG-005 - Remove Letter Hotkeys

---

Letter hotkeys were removed from the global keyboard handler.

The immediate issue was that typing `p` could pause the app. That made normal romaji input unsafe because a learner could accidentally trigger app controls while typing.

Removed behavior:

```txt
p = pause / resume
```

Remaining keyboard behavior:

| Key        | Scope                      | Behavior                                      |
| ---------- | -------------------------- | --------------------------------------------- |
| ArrowLeft  | Global while article open  | Previous token; Shift moves previous sentence.|
| ArrowRight | Global while article open  | Next token; Shift moves next sentence.        |
| Escape     | Global                     | Close modal or open menu.                     |
| Space      | Outside typing input only  | Pause / resume.                               |
| Enter      | Typing input only          | Complete current token if input is correct.   |

This preserves non-letter navigation while preventing normal typed letters from controlling the app.
