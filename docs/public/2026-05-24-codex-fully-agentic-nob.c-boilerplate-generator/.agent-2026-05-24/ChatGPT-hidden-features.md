A00 Scope

---

As of May 23, 2026, the useful way to think about ChatGPT is not "chatbot with text." It is closer to "chat surface plus optional tools." The hidden part is that many tools are not obvious until you ask for them directly, use the `+` menu, use `/` shortcuts, open Settings, or start from a special surface such as Images, Canvas, Projects, GPTs, Apps, Tasks, Codex, Agent, or Pulse.

Availability varies by plan, region, rollout, app version, and workspace settings. OpenAI's docs repeatedly note plan and workspace limits for tools such as apps, data analysis, files, deep research, GPTs, and tasks. ([OpenAI Help Center][1])

C00 Practical discovery workflow

---

Use this as a repeatable experiment whenever you suspect ChatGPT has a hidden capability.

| Step | Action                                            | Prompt                                                                                                                              |
| ---- | ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| 1    | Ask for the right surface                         | "For this task, should I use normal chat, Canvas, Data Analysis, Images, Deep Research, Agent, Codex, a GPT, a Project, or a Task?" |
| 2    | Force tool discovery                              | "List every ChatGPT feature that could improve this workflow. Include features that are not obvious from the chat box."             |
| 3    | Request multiple outputs                          | "Give me 5 variants, each optimized for a different direction. Do not make minor variations."                                       |
| 4    | Ask for an executable artifact                    | "Make this as a self-contained HTML/CSS/JS prototype that I can preview. Avoid external dependencies."                              |
| 5    | Move long work into a persistent surface          | "Turn this into a Project with reusable instructions, source files, and a decision log."                                            |
| 6    | Use files aggressively                            | "Use the uploaded files as source material. Cite which file each claim comes from."                                                 |
| 7    | Use screenshots as input                          | "Analyze this screenshot as if you are doing a design QA pass. Identify layout, accessibility, copy, and interaction problems."     |
| 8    | Use Search when freshness matters                 | "Search the web and cite official sources. Do not rely on memory."                                                                  |
| 9    | Use Deep Research when the answer needs synthesis | "Use Deep Research. First propose the research plan, then produce a cited report."                                                  |
| 10   | Convert repeated work into automation             | "Create a recurring task for this check and notify me only when something changes."                                                 |

D00 Highest-value experiments for your use case

---

For UI and frontend work, start with Images, image inputs, Canvas, executable HTML/CSS/JS prototypes, and screenshot critique. This gives you a loop where ChatGPT can generate a mockup, render or preview code where supported, critique the result, then revise the implementation.

For engineering work, test GitHub connection, Codex, Data Analysis, file uploads, Projects, and custom GPTs. The strongest workflow is: connect repo or upload files, ask for architectural understanding, move the work into a Project, use Canvas for focused edits, and use Codex when you want code changes rather than explanation.

For research and planning, use Search for quick current facts, Deep Research for cited synthesis, Tasks for recurring checks, and Pulse if you have Pro and want proactive daily summaries.

For personalization, configure Memory, Custom Instructions, and project-level instructions separately. Memory is for durable preferences, Custom Instructions are for global behavior, and Project instructions are for task-specific behavior that should override the global defaults inside one project.
