# Option Comparison

The strongest product shape is a controlled combination of:

- Conservative copy-first generator.
- Safe defaults with advanced controls.
- Assumption-driven validation.
- Deterministic fixture-first generation.
- Short block-level explanations.

The comparison rejected a full starter workflow configurator for v1 because target dispatch, run/test/clean commands, static-library generation, and cross-platform branches increase invalid-code risk before the baseline generator is proven.

The comparison also rejected a pure teaching/code-lens tool for v1 because it spends complexity on UI explanation mechanics instead of generated-code correctness.

The selected approach keeps the generated `nob.c` quality as the release gate.
